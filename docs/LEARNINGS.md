# LEARNINGS

## Development Workflow

- One milestone per session.
- Documentation updated after each completed milestone.
- New chat after each milestone.

---

## Coding Rules

- Full paste-ready code only.
- Never partial code.
- One action at a time.
- Explicit execution location.
- Explicit terminal.
- Explicit folder.
- Explicit virtual environment.

---

## Architecture Principles

- Build for production from day one.
- Avoid temporary solutions requiring future rewrites.
- CSV is temporary.
- PostgreSQL is the source of truth.
- Redis stores real-time data only.
- Docker manages infrastructure.

---

## AI Principles

LLMs never participate in live trade execution.

LLMs are used for:

- Research
- Strategy analysis
- Signal explanation
- News interpretation
- Code generation

Trading engine remains deterministic.
